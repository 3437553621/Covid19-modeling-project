# Machine Learning Model Update

## 改进原因

旧版机器学习部分只使用 Linear Regression 和 Random Forest，并且特征主要是少量滞后值与 7 日滚动统计。对 COVID-19 这种存在阶段性政策变化、周周期和补报修正的时间序列来说，特征表达偏弱，导致确诊、死亡和治愈任务的测试集效果不稳定。

## 本次改进

本次在不使用未来信息的前提下重新设计了机器学习预测部分：

1. 增加 1-14 天、21 天、28 天滞后特征，用于捕捉短期惯性和周周期。
2. 增加 3、7、14、28 日滚动均值、中位数、标准差和最大值，用于描述近期趋势与波动。
3. 增加日期特征，包括星期、月份、周序号和星期周期正余弦编码。
4. 增加指数滑动均值、1 日差分、7 日差分和相对 7 日均值比例。
5. 保留 Linear Regression 和 Random Forest，同时新增 Ridge Regression、Extra Trees、Gradient Boosting 和 RollingMean7 基准。
6. 所有特征都只由预测日前已经发生的数据构造，避免未来信息泄露。

## 当前效果

改进后，`outputs/metrics/china_all_ml_metrics.csv` 中保存了最新测试集指标。主要变化如下：

- 每日新增确诊：最佳模型为 RollingMean7，R2 提高到约 0.676；GradientBoosting 的 R2 也达到约 0.623。
- 每日新增治愈：最佳模型为 GradientBoosting，RMSE 降低到约 11.34，MAE 降低到约 7.41。
- 每日新增死亡：MAE 最优模型为 GradientBoosting，约 376.40；R2 仍略低于 0。

死亡人数的 R2 仍然不理想，主要原因是 2023 年 1 月存在极端补报尖峰，例如 2023-01-15 单日死亡新增 59961。这类统计口径修正不是由前几日常规趋势自然演化出来的，普通时间序列模型无法在不引入人工事件变量或未来信息的情况下提前预测。报告中应如实说明该问题，不能为了提高指标而删除或伪造数据。

## 建议写入论文的解释

机器学习模型对每日新增确诊的短期趋势拟合效果较好，说明滞后项和滚动均值能够捕捉疫情传播的惯性特征。治愈数据由于 JHU 在 2021-08-05 后停止可靠更新，可用样本较短，模型效果有所改善但仍存在不稳定。死亡数据受补报和统计口径调整影响明显，RMSE 和 R2 容易被少数极端值主导，因此应结合 MAE 和趋势图综合分析。
